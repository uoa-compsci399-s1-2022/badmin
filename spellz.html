About

Spellz is an engaging proofreading game, which implements the full ReType functionality. This means that Spellz will include existing game mechanics such as scores, combos and hints, while also implementing an active input box.

Select Game

You have the option of choosing the daily challenge or a game within specific category and genre.

The daily challenge by default is on the landing page or you can access it by clicking on the logo or just selecting a difficulty option on its own.

Once the game is played you can also choose to replay again.

To play a game within specific difficulty and genre levels you just need to click on the specific buttons on the upper right corner.

The buttons are yellow to show what was selected by the user.

Start Game

In order to start the game you must click the blurred text.

Active Input Box

You enter the correct spelling of the word you deem to have incorrect spelling in the active input box below the text.

Game Mechanics

Score:

The score accumulates on the basis of combo, hint used and time. In the case user enters wrong correction in input box the score deducts by 30 points.

Combos aside, one correct input increases score by 100 points

Combos:

To add to the combo the user needs to get correct inputs 5 seconds from each other.

By default each user is on combo 1 and gets 100 points for each correct input.

When reaching a combo 2 from combo 1, by entering next correct input within 5 seconds, a user gets an additional 200 points instead of standard 100 points.

The same concept applies from combo 2 to combo 3, as when user enters correct input within 5 seconds of correct answer, the user gets 300 points added on.

The maximum additional points one can gain from combos is 300 points, so a user can keep being at combo 3 by continuously entering correct inputs 5 seconds apart from each other.

Hint:

The hint appears within 30 seconds of game start and user has the option of choosing it at the cost of deducing points as it reveals how many incorrect words are remaining in the text.

Game End

In order to end playing the game the user can press the stop button. Once the button is clicked, the user will be presented a modal giving an overview on how they did and also a functionality to share their results.
